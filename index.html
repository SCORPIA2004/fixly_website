Today a broken streetlight gets reported five ways — çağrı merkezi, CİMER, Beyaz Masa, social media, one-off apps. Citizens never hear back; staff drown in duplicates. Fixly is one pipeline with two surfaces: a citizen app and a staff dashboard, writing to the same backend.

Photo to fixed, in three moves

No signup wall, no forms. The camera is the interface — AI fills in the rest.

  1. 01

    Citizen snaps it

    Anonymous session on first launch — no account. Photo + GPS + an optional one-liner, submitted from the Expo app in about 30 seconds.

  2. 02

    AI triages it

    Hugging Face vision suggests the category in 2–8 seconds. PostGIS checks a 50 m radius — the 12th report of the same pothole joins the existing cluster instead of the queue.

  3. 03

    Staff resolve it

    One inbox: confirm category, assign the department, mark resolved. The citizen watches the status move — İnceleme → Atandı → Çözüldü — instead of wondering if anyone read it.

Built for the workflow, not the form

The reporting form is the easy part. Deduplication and staff triage are the moat.

CV-assisted categories

Vision model proposes the category; staff confirm with one click. Inanimate objects only — no person or plate detection, ever.

PostGIS deduplication

ST_DWithin clustering per category — 50 reports of one pothole become a single work item with a reporter count.

Status transparency

Every report carries a public timeline from submission to resolution. Trust is the feedback loop, not the submission.

Bilingual by default

TR/EN across both surfaces from one shared catalog — formal siz throughout, locale-aware statuses, switchable mid-session.

KVKK from day one

Anonymous citizen sessions, redaction before inference, deletion policy in the docs — privacy is an architecture decision, not a checkbox.

AI-built, in the open

Developed end-to-end with Cursor — ADR-locked decisions, agentic rulesets, and a Cursor SDK script that keeps TR translations in sync.

Two surfaces, one backend

Both apps write to the same Go API. Events flow through an in-process bus to the CV pipeline; reports land in Postgres with PostGIS doing the spatial work.

  • Citizen app — Expo · React Native · anonymous JWT
  • Staff dashboard — Next.js · triage inbox · map view
  • API — Go (masterfabric) · Postgres + PostGIS · Hugging Face inference

Run the whole thing locally

Postgres, Redis and the API in one script. Seeded with Demo Belediyesi — categories, staff users and ~30 reports.

# API + database
cd services/api && ./dev.sh

# Staff dashboard → localhost:3000
cd apps/web && npm run dev

# Citizen app → scan QR with Expo Go
cd apps/mobile && npm start

Full setup, API keys and demo script in docs/.
